refactor: reorganize codebase into modular folder structure

Major refactoring to improve code organization and maintainability by
restructuring the security framework into logical modules.

## Code Reorganization

### New Modular Structure
- **src/core/**: Main worker entry point (moved from src/index.ts)
- **src/bot/**: Bot management module
  - Credential stuffing detector
  - Spam detector
  - Session manager
  - Traffic classifier
  - Adaptive rate limiter
  - Turnstile integration
- **src/rate-limiter/**: Rate limiting module
  - Core rate limiter with sliding window
  - Burst detector for DoS/DDoS protection
- **src/rules/**: WAF rules module
  - WAF rule engine
  - WAF configuration
  - Deserialization patterns
- **src/tracing/**: Request tracing and geolocation
  - Request tracing system
  - Geolocation blocking
  - IP tracking and reputation
- **src/utils/**: Utility functions
  - Debug logging utilities

### Removed Files
- Deleted flat structure files from src/ root:
  - src/index.ts → moved to src/core/index.ts
  - src/rate-limiter.ts → moved to src/rate-limiter/rate-limiter.ts
  - src/burst-detector.ts → moved to src/rate-limiter/burst-detector.ts
  - src/waf.ts → moved to src/rules/waf.ts
  - src/waf-config.ts → moved to src/rules/waf-config.ts
  - src/tracing.ts → moved to src/tracing/tracing.ts
  - src/traffic-classifier.ts → moved to src/bot/traffic-classifier.ts
  - src/turnstile.ts → moved to src/bot/turnstile.ts
  - src/adaptive-rate-limiter.ts → moved to src/bot/adaptive-rate-limiter.ts
  - src/debug.ts → moved to src/utils/debug.ts
